# hooks 目录文档

## 目录功能概述

`hooks` 目录用于存放 PyInstaller 的钩子脚本文件，这些脚本在打包过程中用于处理特定的模块依赖和资源文件。

## 直属文件说明

### 📁 `__pycache__/`
- **功能**: Python 字节码缓存目录
- **说明**: 存放编译后的 `.pyc` 文件，用于提高模块加载速度
- **包含文件**:
  - `hook-local_agent.ui.cpython-39.pyc` - hook-local_agent.ui.py 的 Python 3.9 字节码缓存

### 📄 `hook-local_agent.ui.py`
- **功能**: PyInstaller 钩子脚本
- **用途**: 专门用于处理 `local_agent.ui` 模块的打包依赖
- **主要功能**:
  - 自动检测和包含 `local_agent.ui` 模块所需的隐藏导入
  - 确保 UI 相关的资源文件正确打包到最终的可执行文件中
  - 处理 tkinter 等 GUI 框架的依赖关系

## 钩子脚本工作原理

PyInstaller 钩子脚本在打包过程中自动执行，用于：

1. **依赖分析**: 分析模块的导入依赖关系
2. **资源收集**: 收集模块所需的资源文件（如图片、配置文件等）
3. **路径处理**: 处理模块的运行时路径问题
4. **隐藏导入**: 添加动态导入的模块到打包列表中

## 使用说明

当使用 PyInstaller 打包项目时，这些钩子脚本会自动被调用，无需手动干预。如果添加了新的 UI 组件或依赖，可能需要更新相应的钩子脚本。
